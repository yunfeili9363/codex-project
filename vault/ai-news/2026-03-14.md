### AWS 把 P-EAGLE 推进 vLLM，B200 上推理提速最高 1.69x

- 摘要：AWS 于 2026-03-13 发文介绍 P-EAGLE，并说明其已从 vLLM v0.16.0 起集成。核心点是把 speculative decoding 的 K 个草稿 token 改成单次 forward 并行生成，减少传统 EAGLE 的串行草稿瓶颈；官方给出的 B200 实测提速区间为 1.05x 到 1.69x。
- 为什么值得关注：这是高价值的推理基础设施更新。对做模型服务、Agent 产品和中文技术内容的创作者来说，它提供了一个很好的选题切口：为什么推理优化现在比单纯堆参数更重要，以及 speculative decoding 正在怎样演化。
- 适合内容角度：可以做一期“推理提速不靠换模型，而靠解码算法”的科普，顺带讲清 EAGLE、P-EAGLE、vLLM、B200 之间的关系，并延伸到国产模型部署场景。
- 来源：https://aws.amazon.com/blogs/machine-learning/p-eagle-faster-llm-inference-with-parallel-speculative-decoding-in-vllm/

### Ollama v0.18.0 发布，本地与云模型切换更轻

- 摘要：Ollama 于 2026-03-14 发布 v0.18.0。最值得注意的变化是 cloud models 不再需要先执行 `ollama pull`，改为使用 `:cloud` 标签即可自动连接；同时 `ollama launch claude` 新增对 Claude Code compaction window 的设置支持。
- 为什么值得关注：这不是大而全的版本，但方向很明确：Ollama 正在把“本地模型工具”进一步做成“本地+云”的统一入口。对中文创作者和开发者读者来说，这直接关系到 AI 工作流会不会继续向单一 CLI 聚合。
- 适合内容角度：适合做“本地 AI 工具开始吃掉云入口”的观察稿，或者做一期实操内容：如何用 Ollama 同时管理本地权重、云模型和 Claude Code 工作流。
- 来源：https://github.com/ollama/ollama/releases/tag/v0.18.0

### ComfyUI v0.17.0 更新 Qwen Image 与 Flux 2 Klein 工作流

- 摘要：ComfyUI 于 2026-03-13 发布 v0.17.0。更新重点包括新的 modular assets 架构、为 Qwen Image 增加 pre-attention 与 post-input patches、支持 Flux 2 Klein 的 KV cache 工作流，并加入 Reve Image nodes。当天晚些时候又发了 v0.17.1 小版本。
- 为什么值得关注：对偏创作向的中文受众，这比泛泛的模型新闻更有用。它直接影响图像工作流、节点生态和新模型接入速度，也说明 ComfyUI 仍在快速吸收新一代图像模型能力。
- 适合内容角度：适合做“ComfyUI 为什么仍是创作者工作台”的选题，重点拆 Qwen Image、Flux 2 Klein、Reve Image 三条线分别能带来什么新玩法。
- 来源：https://github.com/Comfy-Org/ComfyUI/releases/tag/v0.17.0

### llama.cpp b8333 修复 Qwen3.5 在 Apple Silicon 上的性能回归

- 摘要：llama.cpp 于 2026-03-14 发布 b8333。此次更新集中在 Gated Delta Net 路径优化，官方说明其修复了 Qwen3.5-9B 在 Metal、M4 Max 上约 39% 的性能回归，并新增 `--fused-gdn [on|off|auto]` 开关，方便控制 fused GDN 行为。

- 适合内容角度：可以做一期“为什么同一个模型在 Mac 上忽快忽慢”的硬核科普，借此解释 Metal、内存访问模式、kernel 优化和开源推理栈的关系。
- 来源：https://github.com/ggml-org/llama.cpp/releases/tag/b8333
