# Bridge 知识库方案

这套结构适合你现在的目标：把零散素材逐步沉淀成可筛选、可判断、可转化的中文笔记库。

## 推荐目录

- `inbox/`
  - 临时收件箱。先记，先存，不做结构化要求。
- `待整理/`
  - 从原始素材到正式笔记之间的加工区。
- `知识库/外部内容/`
  - 外部来源归档，例如 X、视频、网页、播客、文章。
- `知识库/人物/`
  - 人物卡。记录你要跟进、观察、合作、学习的人。
- `知识库/产品/`
  - 产品卡。记录可卖、可研究、可对比的产品。
- `知识库/场景/`
  - 场景卡。记录用户问题、使用时机、触发条件、购买上下文。
- `知识库/可销售性/`
  - 可销售性评估。把人物、产品、场景连起来做判断。
- `80-面板/`
  - Obsidian 查询面板，例如 Dataview 汇总页。
- `90-模板/`
  - 新建笔记时直接复制使用的模板。
- `archives/`
  - 已完成归档、不常改动的旧内容。

## 最小工作流

1. 素材先进 `inbox/` 或 `待整理/`
2. 整理后放入 `知识库/外部内容/`
3. 从素材里拆出 `人物 / 产品 / 场景`
4. 当三者关系逐渐清楚时，新增一张 `可销售性` 评估卡
5. 在 `80-面板/Dataview 筛选面板.md` 里统一查看优先级

## 命名建议

- 人物卡：`人物-姓名.md`
- 产品卡：`产品-名称.md`
- 场景卡：`场景-问题或时机.md`
- 可销售性：`评估-人物-产品-场景.md`

## 字段约定

为了让 Dataview 稳定工作，这套模板统一使用这些字段：

- `note_type`：`person / product / scenario / sellability`
- `name`：笔记主名称
- `status`：跟进状态，例如 `idea / tracking / active / paused / archived`
- `people`、`products`、`scenarios`：用于互相链接
- `overall_score` 或 `sellability_score`：用于排序

## 使用方式

- 最省事的方式：在 `90-模板/` 里复制一份模板到目标目录，再改标题和字段。
- 如果你启用了 Templater，也可以直接把这些模板当作新建模板来源。

## 先从哪里开始

如果你现在资料还很散，建议先只做三件事：

1. 每条外部内容先归档到 `知识库/外部内容/`
2. 每看完一条内容，至少拆出 1 张人物卡或产品卡
3. 只有当“谁买什么、在什么场景下买”逐渐明确时，再写可销售性评估
